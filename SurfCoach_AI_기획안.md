# 🏄 SurfCoach AI — 기획안

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| **서비스명** | SurfCoach AI |
| **한 줄 정의** | AI가 서핑 자세를 분석하고, 전문가 수준의 피드백을 제공하는 코칭 앱 |
| **타겟 유저** | 입문~중급 서퍼 (독학러, 레슨비 부담층) |
| **핵심 가치** | 강사 없이도 자세 교정 가능 → 레슨 비용 절감 |
| **기획자 배경** | 서핑 강사 5년 + 해양 구조대 경력 → 도메인 지식 기반 설계 |

## 2. 핵심 기능 정의

### 🎯 MVP (꼭 있어야 할 기능)

```
① 영상 업로드 → 자세 분석
   - 사용자가 서핑 영상 촬영 후 업로드
   - YOLO로 신체 키포인트 감지
   - 테이크오프(Take-off), 패들링, 스탠스 분석

② 자세 점수 + 피드백 리포트
   - 각 동작별 점수 (0~100)
   - 잘못된 자세 → 텍스트/이미지로 교정 가이드
   - 관절별 색상 오버레이 이미지 제공

③ 개인 히스토리 관리
   - 날짜별 분석 기록 저장
   - 성장 그래프 시각화
```

### 🚀 고도화 기능 (이후 단계)

```
④ 레퍼런스 포즈 매칭
   - 이상적인 자세 레퍼런스 사진 등록
   - 사용자 자세 vs 레퍼런스 유사도 비교
   - 동작 자동 분류 정확도 향상

⑤ 음성 피드백
   - 분석 결과를 TTS로 음성 안내
   - 영상 보면서 귀로 피드백 수신

⑥ 실시간 분석 (라이브 모드)
   - 해변에서 스마트폰 거치 후 실시간 코칭

⑦ 위험 자세 경보
   - 낙수 위험 자세 감지 → 안전 경고
   - (구조대 경력 활용 포인트!)
```

## 3. 서핑 자세 분석 기준 — 도메인 지식 적용

> 이 부분이 경쟁 서비스와의 진짜 차별점

### 분석할 핵심 동작 3가지

#### ① 테이크오프 Take-off (구 팝업 Pop-up)
```
3단계 자동 감지 (body metrics 기반):

1단계 — 푸쉬(Push)
  - 양손이 갈비뼈(어깨~엉덩이 사이) 옆에 위치
  - 팔을 완전히 펴서 체중을 뒤로 이동 → 테일 양력 발생
  - 팔을 구부리면 중심 불안정

2단계 — 발 끌어오기(Pull & Squat)
  - 무릎 각도 90~130도 유지
  - 시선이 바닥을 향하면 중심을 잃고 낙수 → 핵심 체크
  - 충분히 앉은 자세에서 일어서야 안정적

3단계 — 일어서기(Stand Up)
  - 일어나는 동안 시선을 파도 진행 방향으로 고정 → 핵심 체크
  - 무릎 각도 90~120도 유지
  - 상체가 과도하게 앞으로 숙여지지 않도록
```

#### ② 스탠스 (Stance)
```
체크포인트:
- 발 간격 (어깨너비 기준, 이상: 1.0~1.5배)
- 무게중심 (앞발 60%, 뒷발 40%)
- 좌우 무릎 각도 균형
- 어깨 방향 (파도 진행방향과 각도)
```

#### ③ 패들링 (Paddling)
```
올바른 패들링 원리:
- 팔꿈치를 수직으로 물속 깊이 넣고 손바닥으로 물을 뒤로 밀어냄
  (스케이트보드에 엎드려 손바닥으로 바닥을 미는 느낌)
- 최대한 많은 물을 뒤로 밀어내야 추진력 발생
- 물을 퍼올리는 동작(팔꿈치 수직 미유지)은 보드를 뒤뚱거리게 만듦

YOLO 측정 체크포인트:
- 몸통 좌우 대칭 (보드 중심 유지)
- 팔 뻗음 정도 (충분히 앞으로 뻗어야 깊이 입수 가능)
- 좌우 스트로크 균형
- 머리 위치 (과도한 들기 방지)
```

## 4. 기술 스택

### 전체 아키텍처

```
[사용자 스마트폰]
      ↓ 영상 업로드
[React Native 앱 (프론트엔드)]
      ↓
[FastAPI 백엔드 서버]
      ↓                    ↓
[YOLOv8-pose 분석]    [LLM 피드백 생성]
  17개 관절 추출          Gemini 2.5 Flash
  3단계 자동 감지    →   자연어 교정 가이드
  레퍼런스 매칭
      ↓
[오버레이 이미지 생성]
      ↓
[SQLite DB 저장 → 히스토리]
      ↓
[사용자에게 결과 전달]
```

### 상세 기술 스택표

| 파트 | 기술 | 역할 | 완성 여부 |
|------|------|------|-----------|
| **프론트엔드** | React Native | iOS/Android 앱 | 🔲 예정 |
| **백엔드** | FastAPI (Python) | API 서버 | ✅ 완성 |
| **자세 감지** | YOLOv8-pose | 17개 관절 키포인트 추출 | ✅ 완성 |
| **자세 분석 로직** | Python + NumPy | 각도/거리 계산, 3단계 감지 | ✅ 완성 |
| **피드백 생성** | Gemini 2.5 Flash | 자연어 교정 문장 (Claude/GPT-4o 교체 가능) | ✅ 운영 중 |
| **LLM 추상화** | Factory 패턴 | Claude / GPT-4o / Gemini / Gemma4 교체 가능 | ✅ 완성 |
| **시각화** | OpenCV | 관절 오버레이, 점수별 색상 표시 | ✅ 완성 |
| **레퍼런스 매칭** | YOLO + 코사인 유사도 | 이상 자세 vs 사용자 자세 비교 | 🔲 구현 예정 |
| **데이터베이스** | SQLite → PostgreSQL | 분석 기록 저장 | ✅ SQLite 운영 중 |
| **음성 피드백** | gTTS / Whisper | TTS 음성 안내 | 🔲 고도화 단계 |
| **클라우드** | AWS or GCP | 배포 환경 | 🔲 예정 |

## 5. 개발 로드맵

```
✅ 1단계 — 핵심 백엔드 완성
   └─ YOLOv8-pose 서핑 영상 자세 분석 엔진
   └─ 테이크오프 3단계 자동 감지 (body metrics 기반)
   └─ 스탠스 / 패들링 분석 로직
   └─ FastAPI 서버 (업로드 → 분석 → 결과 반환)
   └─ LLM 추상화 (Gemini 2.5 Flash 운영 중)
   └─ 관절 오버레이 이미지 생성
   └─ SQLite 히스토리 저장

🔄 2단계 — 레퍼런스 포즈 매칭 (진행 중)
   └─ 이상적 자세 레퍼런스 사진 라벨링
   └─ YOLO로 레퍼런스 키포인트 추출 및 저장
   └─ 사용자 프레임 vs 레퍼런스 유사도 비교
   └─ 동작 자동 분류 정확도 향상

🔲 3단계 — 프론트엔드 (React Native)
   └─ 영상 업로드 화면
   └─ 분석 결과 화면 (점수 + 오버레이 이미지)
   └─ 히스토리 화면 (성장 그래프)

🔲 4단계 — 고도화
   └─ 음성 피드백 (TTS)
   └─ 실제 서퍼 베타 테스트
   └─ 피드백 반영 후 정식 출시
```

## 7. LLM 모델 추상화 설계

> 개발 버전에서는 모델을 쉽게 교체하고,
> 배포 버전에서는 최종 선정 모델만 고정해서 사용

### 폴더 구조

```
backend/
└── llm/
    ├── base.py          ← 공통 인터페이스 정의
    ├── claude.py        ← Claude API 연동
    ├── gpt4o.py         ← GPT-4o API 연동
    ├── gemini.py        ← Gemini API 연동 (현재 운영)
    ├── gemma4.py        ← Gemma4 로컬 연동
    └── factory.py       ← 환경변수 보고 모델 자동 선택
```

### 환경변수로 모델 전환

```bash
LLM_PROVIDER=gemini   # 현재 운영 중 (Gemini 2.5 Flash)
LLM_PROVIDER=claude   # Claude API로 전환
LLM_PROVIDER=gemma4   # 로컬 배포 버전으로 전환
```

> ✅ 핵심 장점: 모델이 바뀌어도 나머지 코드는 전혀 수정할 필요 없음

## 8. 현재 완성된 백엔드 API

```
POST /analyze
  - 서핑 영상 업로드
  - action: takeoff | stance | paddling
  - 반환: 점수, 문제점, 오버레이 이미지 URL, LLM 피드백

GET /history
  - 분석 기록 목록 조회

GET /history/{id}
  - 단건 상세 조회
```

프론트엔드 개발자는 위 API에 바로 연동하면 됩니다.
